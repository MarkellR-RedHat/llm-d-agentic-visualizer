The Solution

KV Cache-Aware Routing
with EPP Scoring

llm-d's Endpoint Picker Policy (EPP) hashes the system prompt prefix and routes subsequent calls to pods that already hold a warm KV cache. The result -- near-zero prefill for calls 2 through 7.

1 Prefix Hashing

EPP hashes the system prompt + tool definitions into a prefix key (e.g., 0xa3f7c2e1). This key identifies the cacheable prefix shared across all LLM calls in a session.

2 Pod Scoring

Each vLLM pod receives a composite score: 60% cache match + 20% queue depth + 20% GPU load. The pod with the highest score wins the request.

3 Warm Routing

Calls 2-7 route to the pod with a warm KV cache, skipping up to 92% of prefill computation. TTFT drops from 350ms to 55ms.

Multi-Agent

Cross-Agent Cache Sharing
